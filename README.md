# RAG CI/CD Pipeline с проверкой на галлюцинации (Ragas)

## Описание проекта
Этот репозиторий содержит RAG-системы на базе LlamaIndex, ChromaDB и Hugging Face API. 
Основная цель — автоматизированная проверка качества ответов LLM, выявление галлюцинаций и интеграция этих проверок в CI/CD пайплайн.

## Используемые метрики (Ragas)
Для оценки качества используются следующие метрики:

1. **Faithfulness (Верность)**: Оценивает, насколько ответ согласуется с предоставленным контекстом.
   - *Цель*: Выявить галлюцинации (факты, выдуманные моделью).
   - *Порог*: ≥ 0.7
2. **Answer Relevance (Релевантность ответа)**: Оценивает, насколько ответ непосредственно отвечает на заданный вопрос.
   - *Порог*: ≥ 0.7
3. **Context Recall (Полнота контекста)**: Оценивает, содержит ли retrieved контекст всю информацию, необходимую для формирования золотого ответа.
   - *Порог*: ≥ 0.7

## Установка и запуск локально

1. Клонировать репозиторий.
2. Установить зависимости:
   ```bash
   pip install -r requirements.txt
   
3. Экспортируйте токен Hugging Face
export HF_TOKEN="ваш_токен_hf"

4. Запустить тесты:
pytest tests/test_ragas_eval.py -v -s

5. Результаты сохранятся в файле ragas_results.json.
